# IM-Scholar 白皮書：學術文獻 RAG 系統之理論與實踐

### 1. 領域知識轉化之難點
資管領域（IM）文獻具備「高抽象性」與「多重表徵」的特點。例如，一個「TAM（科技接受模型）」可能出現在流程圖、統計表或純文字中。傳統向量化難以精確捕捉公式與變數間的邏輯關係，導致數位表徵（Representation）出現語義稀釋。

### 2. 幻覺 (Hallucination) 解決方案
IM-Scholar 透過以下機制抑制幻覺：
* **引用來源對齊：** 強制 LLM 在生成每一段論點時，必須標註檢索結果中的 `Source ID`。
* **檢索閾值過濾：** 凡是 Reranking 評分低於 0.7 的片段一律不進入生成視窗，寧可回答「文獻未提及」也不編造答案。

### 3. 創新性：冷啟動與術語對齊
針對新興資管議題（如：生成式 AI 在企業的落地），系統透過 **Hybrid Search** 解決冷啟動問題，利用傳統 BM25 關鍵字檢索確保新術語能被精確定位，再由語義檢索補充背景知識。
